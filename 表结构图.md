# 数据库表结构图

> 基于《需求分析说明书》，18 张表，按业务流分层展示，关系一目了然。

## 一、核心业务流总览（一条线看懂系统）

```
                    ┌─────────────────────────────────┐
                    │         ① 学期 SEMESTER          │
                    └──────────────┬──────────────────┘
                                   │ 本学期开哪些课?
    ┌──────────────────┐          ▼
    │ ② 课程 COURSE    │──────────────────────┐
    │   (课程目录)      │ 1:N                  │ 1:N
    │   ┌─────────────┐│                      │
    │   │LevelRule    ││   ③ 开课 COURSE_OFFERING ◄─── ⑨ 教师 TEACHER
    │   │(本/研学分)   ││      (教学班,具体某学期)         (授课人)
    │   └─────────────┘│       │
    └──────────────────┘       │
                               ├──► ④ 选课 ENROLLMENT ◄─── ⑩ 学生 STUDENT
                               │      (学生选了哪个班)        (选课人)
                               │
                               ├──► ⑤ 排课 SCHEDULE ────► ⑦ 教室 CLASSROOM
                               │      (何时何地上课)         (上课地点)
                               │           │
                               │           ├──► ⑧ 调课 SCHEDULE_CHANGE
                               │           │      (时间/地点变更)
                               │           │
                               │           └──► ⑰ 考勤 ATTENDANCE ◄── 学生
                               │                  (每次到课情况)
                               │
                               ├──► ⑥ 成绩体系
                               │      ├── GRADING_TEMPLATE (评分模板,预设权重)
                               │      ├── GRADE            (总评成绩)
                               │      └── GRADE_ITEM       (成绩子项:平时/期中/期末...)
                               │
                               └──► 扩展业务
                                      ├── ⑮ 毕业设计 THESIS ◄── 教师(指导+评阅)
                                      └── ⑯ 项目设计 PROJECT ◄── 教师(指导)
```

> **一句话概括**：学期里，教师为课程开设教学班 → 学生选班 → 教务排时间教室 → 日常考勤 → 期末评分。毕业设计和项目独立管理。

## 三、按业务场景拆分的子图

### 子图 A：选课流程（谁选了谁的什么课）

```mermaid
erDiagram
    SEMESTER {
        char11 sem_id PK "学期编号"
    }
    COURSE {
        char8 course_id PK "课程编号"
        varchar course_name "课程名称"
        char3 course_type "理论|实验|实践"
    }
    TEACHER {
        char8 teacher_id PK "教师编号"
        varchar teacher_name "姓名"
    }
    COURSE_OFFERING {
        char12 offering_id PK "开课编号"
        char8 course_id FK "课程"
        char8 teacher_id FK "教师"
        char11 sem_id FK "学期"
        int max_capacity "人数上限"
    }
    STUDENT {
        char10 stu_id PK "学号"
        varchar stu_name "姓名"
        char4 degree "本科|硕士|博士"
    }
    ENROLLMENT {
        int enroll_id PK "选课编号"
        char10 stu_id FK "学号"
        char12 offering_id FK "开课"
        timestamp enroll_time "选课时间"
    }

    SEMESTER ||--o{ COURSE_OFFERING : "本学期开课"
    COURSE ||--o{ COURSE_OFFERING : "开设教学班"
    TEACHER ||--o{ COURSE_OFFERING : "授课"
    COURSE_OFFERING ||--o{ ENROLLMENT : "被选"
    STUDENT ||--o{ ENROLLMENT : "选课"
```

### 子图 B：排课流程（什么课在何时何地上课）

```mermaid
erDiagram
```

## 六、关系强度分类

| 强度 | 关系 | 说明 |
|:--:|------|------|
| ⭐⭐⭐ | CourseOffering ← Enrollment → Student | **核心：选课** | 
| ⭐⭐⭐ | CourseOffering → Schedule → Classroom | **核心：排课** |
| ⭐⭐⭐ | Student + CourseOffering → Grade → GradeItem | **核心：成绩** |
| ⭐⭐ | Course → CourseLevelRule | 本研学分区分 |
| ⭐⭐ | Schedule → Attendance ← Student | 日常考勤 |
| ⭐⭐ | Schedule → ScheduleChange | 调课管理 |
| ⭐ | Student → Thesis | 毕业设计(1:1) |
| ⭐ | Student → Project | 项目设计 |
| ⭐ | Department → (Student, Teacher, Course, Major) | 基础归属 |
